# GitHub Trending V5 输出规范

## 分组标题
统一使用二级标题，例如：
- `## Trending`
- `## Rust`

## 仓库条目固定格式
- `**序号. [owner/repo](repo_url) - 🌟 star数 - 语言**`
- `- 仓库说明（中文翻译）：...`
- `- 作用详解：...`

## 约束
- 不得保留 llms.txt 原始三行结构。
- 仓库说明必须是中文表达。
- 作用详解必须尽量具体，避免空泛价值判断。
- 禁止只写：
  - 提升效率
  - 降低门槛
  - 适用于多种场景
  - 赋能开发
- 应尽量写清：
  - 处理什么对象
  - 用于什么任务
  - 服务什么场景
  - 面向什么用户
- 若信息不足，明确写“信息不足”。
